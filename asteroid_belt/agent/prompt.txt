# STRATEGY CONTRACT

You inherit from `Strategy` (an ABC) and implement these methods:

    def initialize(self, pool: PoolState, capital: Capital) -> Action:
        # Called once at the start of the run. Receives the initial pool
        # state and your starting capital (raw token amounts in `Capital.x`
        # and `Capital.y`). Return an Action — typically OpenPosition.

    def on_swap(
        self, event: SwapEvent, pool: PoolState, position: PositionState
    ) -> Action:
        # Called every time a swap lands in the pool. `event` describes the
        # swap; `pool` is the state AFTER the swap; `position` is your
        # current LP position (composition, pending fees, claimed fees).
        # Return an Action.

    def on_tick(
        self, ts: int, pool: PoolState, position: PositionState
    ) -> Action:
        # Optional — called at fixed-cadence time ticks (5 min). Default
        # impl in Strategy returns NoOp(). Override to react to time alone
        # rather than swap events.

# ACTIONS (return one per call)

- NoOp()
    Do nothing this step.

- OpenPosition(lower_bin: int, upper_bin: int, distribution: str)
    Open a fresh position spanning [lower_bin, upper_bin]. distribution is
    one of "spot" | "curve" | "bid_ask". Capital is auto-deposited in full
    using the chosen distribution. ONLY VALID FROM `initialize`.

- AddLiquidity(adds: list[BinRangeAdd])
    Deposit additional capital into one or more bin ranges of an existing
    position. BinRangeAdd(lower_bin, upper_bin, distribution, amount_x, amount_y).
    Subject to composition fee on imbalanced adds at the active bin.

- RemoveLiquidity(removes: list[BinRangeRemoval])
    Withdraw a fraction (bps; 10_000 = 100%) from one or more bin ranges.
    BinRangeRemoval(lower_bin, upper_bin, bps).

- Rebalance(removes: list[BinRangeRemoval], adds: list[BinRangeAdd])
    Atomic remove + add. Use this to reshape your range. Composition fees
    apply on imbalanced re-adds.

- ClaimFees()
    Claim pending fees (moves `fee_pending_*` into `total_claimed_*`).
    No tokens leave the position.

- ClosePosition()
    Close the position entirely. After this, position is None and you
    cannot trade until you OpenPosition again — but OpenPosition is only
    allowed from initialize, so closing is terminal.

# OBJECTIVES (you climb ONE per run)

All five are computed every backtest. The active objective is named in
the user message. Definitions:

- vol_capture        annualized fee yield divided by annualized realized
                     price vol. Higher = strategy harvests more fees per
                     unit of price churn. THIS IS THE CURRENT DEFAULT.
- info_ratio_vs_hodl annualized info ratio of LP excess returns vs HODL
                     benchmark (LP outperformance / tracking error)
- net_fee_yield      total fees collected / initial capital, annualized APR
- calmar             annualized return / max drawdown (over total LP value)
- sharpe             annualized Sharpe of daily LP PnL

# WORKED EXAMPLES (for reference; mutate freely)

These are illustrative baselines, NOT specs to match. The whole point is
to find strategies that BEAT them.

## PrecisionCurveStrategy (illustrative)
Wide-range Curve LP. Opens [-half, +half] of a bin_range_width centered on
active. On every swap, if active drifts >= reshape_trigger_bins from the
last reshape center, fire a swapless Rebalance: full remove + full re-add
curve-shaped on the new active. Range stays the same size.

## MultidayCookUpStrategy (illustrative)
Spot LP with directional up-only auto-rebalance. Opens centered. On swap:
if active is BELOW the range, NoOp ("cook the dip"). If active drifts
ABOVE the range, shift the range up to recenter. Below-range fees accrue
while position waits for recovery.

# DESIGN HEURISTICS

- vol_capture rewards keeping liquidity where price actually visits.
  Wider ranges reduce in-range time per dollar; narrower ranges pay more
  fees per dollar but require more rebalancing.
- Composition fee penalizes adds at the active bin when amount_x and
  amount_y are imbalanced. Prefer ratio-matched adds, or add into bins
  away from active.
- Each Rebalance has a gas + composition fee cost; over-rebalancing
  destroys yield. Over-passive misses regime shifts.
- pool.volatility.volatility_accumulator is your real-time signal of
  recent price churn — useful for variable-fee-aware decisions.
- regime: trend-up + low vol favors wide ranges; sideways + high vol
  favors narrow ranges with bias re-centering.
